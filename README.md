## Практика

`git` можно использовать для любых задач!


### Легенда

Вы с коллегой вдвоем пишете книгу о котах.
Вы уже создали три главы.

Теперь вы паралелльно работаете над новыми главами и редактируете существующие.
Вы - отвечаете за редактуру книги и принятие всех ключевых решений.
Следовательно, вам придется присматривать за работой коллеги.


Перед выполнением задания:

1. Необходимо создать на scm.x5.ru в персональном пространстве репозиторий git_02.
2. Загрузить в него файлы из папки book в архиве с заданием. Для инициализации репозитория используйте локально команду git init. А затем выполните git push.



Ваша задача:

1. Создать четвертую главу
1.1 Создать новую ветку `chapter-4` и перейдем на нее
1.2 Дописать новую главу про котов в файл `book/4.md` (можно воспользоваться сайтом http://www.catipsum.com/)
1.3 Сделать `push` вашей ветки в удаленный репозиторий
1.4 Создать "запрос на слияние изменений" в вашем удаленном репозитории
1.5 Теперь оставим ее там полежать, чтобы ваш коллега успел посмотреть изменения

2. Отредактировать третью главу
2.1 Создать новую ветку `chapter-3-edit` и перейдем на нее
2.2 Заметим, что в третьей главе два абзаца повторяются. Упс!
2.3 Удалим второй абзац
2.4 Заменим его на фразу `Cats are awesome!`
2.5 Сделаем коммит
2.6 Сольем ветку `chapter-3-edit` с `master`

3. Проверим текущее состояние
3.1 Вернемся на ветку `master`. Посмотрим `log` и `status`
3.2 Проверим, что четвертой главы нет (и не должно быть в `master`)
3.3 Проверим, что третья глава отредактирована

4. Вытянем изменения коллеги

Наш коллега хорошо поработал! Что он сделал?

- Добавил вводную главу `0.md`
- Отредактировал главу `2.md`: разбил один из абзатцев, добавил под-заголовок
- Отредактировал главу `3.md`: тоже заметил и удалил повторный абзац, заменил его на фразу `Cats are pretty.`
